The basic ESP32-S3 exact-version consumer and the repository's existing
firmware environment also compiled successfully with the same installed
PlatformIO/Espressif toolchain. No firmware was uploaded or run. The static
figures above describe the complete benchmark firmware image, not MicroWorld in
isolation, and they do not establish runtime heap, stack, or cycle behavior.

The PlatformIO native consumer resolved MicroWorld 0.1.0 correctly but could
not compile because this Windows host has no GNU `g++` on `PATH`. Standalone
CMake/MSVC and Clang no-exceptions/no-RTTI host compilation passed, so this is a
remaining consumer-environment prerequisite rather than an ESP32 or framework
compile failure.
