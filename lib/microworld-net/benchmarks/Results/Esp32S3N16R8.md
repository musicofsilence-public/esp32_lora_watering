## Verified corrected-source result

The Core+Memory+Net consumer was built with PlatformIO using the checked-in
`esp32-s3-net` environment. Build, library-dependency, and cache directories
were redirected to a clean temporary evidence directory so no generated file
entered the repository.

```powershell
$env:PLATFORMIO_BUILD_DIR = '<temporary-evidence-root>\pio-build'
$env:PLATFORMIO_LIBDEPS_DIR = '<temporary-evidence-root>\pio-libdeps'
$env:PLATFORMIO_CACHE_DIR = '<temporary-evidence-root>\pio-cache'
pio run -d lib/microworld/tests/consumer -e esp32-s3-net
```

The first correction-pass attempt encountered a transient GCC 15.2.0 internal
compiler error in the ESP-IDF vendor file `esp_lcd_panel_rgb.c`. A second clean,
isolated build completed successfully in 81.90 seconds without changing
MicroWorld or vendor source.

## What this evidence does and does not establish

- The corrected Core+Memory+Net consumer compiled and linked for ESP32-S3;
  the figures above are compile-only image evidence.
- No firmware upload, radio transmit, runtime timing, stack high-water mark,
  heap measurement, or physical-hardware behavior was performed or recorded
  in either session.
- Net does not include a real transport in this milestone; the ESP32 image
  would only prove the byte-I/O contract compiles on the target, not that any
  radio, UART, or network hardware is initialized or exercised.
