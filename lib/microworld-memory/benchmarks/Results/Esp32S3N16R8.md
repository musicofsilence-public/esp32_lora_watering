## First attempt and correction

The first compile attempt failed because PlatformIO forwarded C++-only
`-fno-rtti` and pedantic/error flags to ESP-IDF C sources. SDK C extensions and
the empty generated C translation unit then became errors. No MicroWorld
production source failed.

The flags were moved to the `src` ESP-IDF component target that instantiates the
representative Memory C++ templates. A clean reconfigure then passed, and the
final incremental rerun also passed. The generated compile command confirmed
`gnu++17`, strict warnings, the narrow `std::aligned_storage` deprecation
suppression, exceptions disabled, and RTTI disabled for
`MemoryEsp32Main.cpp`.

No hardware execution is needed or authorized for this compile gate. Target
runtime evidence remains blocked until explicit upload/run authorization.
